# CLAUDE.md — карта проекта для агента

Тренажёр ПДД Израиля: браузерная игра (TypeScript + Vite + Canvas), игрок
проезжает сценарии, построенные из 144 вопросов теоретического экзамена
(источник isradrive.info). Спецификация для людей — в README.md.

## Команды

```bash
npm test               # vitest, ~217 тестов — ДОЛЖНЫ быть зелёными после любой правки
npm run lint           # eslint; `as`-касты ЗАПРЕЩЕНЫ (assertionStyle: never)
npx tsc --noEmit       # строгие типы
npm run dev            # Vite на :5173
npm run build          # tsc + сборка в ./docs (GitHub Pages)
npm run shot -- 0700 [--drive N|--finish|--mobile]  # скриншоты сцен (см. .claude/skills/run)
```

Процесс — TDD: сначала тест, потом код. UI-тексты — на русском.

## Карта модулей

```

                         израильских знаков), LightState, Task, FailReason

                         (прямо/повороты дугами/разворот/кольцо), isOnRoad, exitedVia

src/ui/{Input,Hud}.ts    клавиатура/кнопки; шапка, панель вопроса+(i), оверлей

                         разрешимости КАЖДОЙ задачи (tests/tasks.test.ts)
```

## Соглашения по координатам и геометрии

- x — на восток, y — на юг (canvas); heading 0 = +x. Подъезд 'S' — игрок
  снизу, едет на север (heading −π/2). Игрок почти всегда спавнится с 'S'.
- `LANE_W = 4.5` (шире реальной: радиус манёвра машины ~3.5 м). Полосы
  подъезда: `lanes[Dir] = {in, out}`; индекс полосы 0 = ПРАВАЯ по ходу.
- Стоп-линия — 1 м до конфликтной зоны; выезд засчитывается за
  `EXIT_THRESHOLD` (12 м) от края зоны по центру машины.
- Повороты — дуги, касательные к осям полос: правый R=4.5, левый R=5.5.
  Разворот (`uturn`) реализован только с 'S' и требует широкой дороги —
  в сцене задавать `lanes: {S:{in:2,out:2}, N:{in:2,out:2}}`.
- Углы тротуаров скруглены «подушками» CURB_PAD=1.5 в isOnRoad — не убирать,
  иначе хвост кузова цепляет угол на поворотах.
- Кольцо: полотно — круг (outerRadius), островок — не дорога; NPC на кольце
  задаётся `ring: {fromAngleDeg, toSide}`, движение — угол atan2 УБЫВАЕТ.

## Модель приоритетов (ядро игровой механики)

- У каждого участника `order` (0 = едет первым); порядок берётся из
  ПРАВИЛЬНОГО ОТВЕТА исходного вопроса — движок его не вычисляет, а проверяет.
- NPC с order < игрока должны проехать до него; въезд игрока в конфликтную
  зону раньше → fail 'priority'. NPC с order > игрока ждут у своих стоп-линий.
- «Подъехали одновременно»: очередь заморожена (`queueStarted`), пока игрок
  не приблизился (<14 м до стоп-линии; для narrow — <25 м до зоны). Без этого
  NPC успевали проехать до подъезда игрока и задачи решались сами.
- NPC.cleared = его OBB вышел из конфликтной зоны; done (конец пути) =
  деспавн (не рисуется, не сталкивается).
- Сужение (narrow): конфликтная зона расширена на ±8 м (диагонали
  перестроения); физическое полотно — NARROW_HALF=2.3, одна нитка по x=0.

## Прочие правила движка

- Тормоз в ПОКОЕ включает задний ход (наследие parking-game) — в тестах и
  скриптах «стоять» = не жать ничего.
- Car.ts + car.test.ts — ОБЩИЕ файлы с ../parking-game и ../drive-town
  (синхронизируются копированием). Расширения: CarInput.handbrake (Space —
  ручник с заносом; обычный тормоз теперь ТОЛЬКО S/↓) и CarOptions.steerRate
  (здесь не используется — руль мгновенный, как раньше). Без ручника физика
  бит-в-бит старая.
- Знак «стоп»/жд: нужна полная остановка (v<0.12) в стоп-зоне до пересечения
  линии. Мигающий жёлтый: пересечение линии при v>6 м/с → fail.
- Пешеходы курсируют вечно (пауза 3.4 с на тротуаре); нарушение — ВЪЕЗД на
  зебру, когда пешеход на проезжей части (transition, не непрерывно).
  Несколько пешеходов в сцене — задержки подбирать так, чтобы их «рейсы»
  шли внахлёст и оставалось окно (см. задачу 0190).
- Светофор: fail при пересечении линии на red/red-yellow/yellow/red-flashing.
  `busArrow: true` — доп. секция «жёлтая стрелка для ОТ» (рендер).

## Авторинг задачи (чек-лист)

1. id — номер вопроса isradrive; текст/ответ должны быть в
   `src/tasks/qbank.json` (подмножество `data/questions.json` по
   `data/selection.json`; при добавлении id — дописать в оба).
2. Сцена в соответствующий `src/tasks/<категория>.ts`; порядок проезда — из
   правильного ответа; цвета/типы NPC — как на картинке вопроса
   (`https://he.isradrive.info/pic/<pic>`, pic указан в qbank).
3. `npm test` — tasks.test.ts прогонит автопилот; задача обязана проходиться.
4. Визуально: `npm run shot -- <id> --finish`.

## Данные и знаки

- `scripts/fetch-questions.sh` + `parse-questions.mjs` → `data/questions.json`
  (полная база AB, 1264 вопроса). В бандл идёт только qbank.json.
- Знаки: `public/signs/<номер>.png`, вырезаны из спрайта
  `isradrive.info/znaki/znaki.png` (78×63/тайл). Ограничения скорости с
  нестандартным числом — генерируются `node scripts/gen-speed-signs.mjs`:
  `426-30/40`, `427-30/40`
  (исходник нарисован с «50», круг в тайле смещён влево — рантайм-оверлей
  не работает).
- Отладка конкретной задачи в браузере: `http://localhost:5173/?task=0700`.
