# Тренажёр ПДД Израиля

Браузерный тренажёр по правилам дорожного движения Израиля. Вместо выбора ответа из четырёх вариантов игрок **проезжает ситуацию сам**: подъезжает к перекрёстку, читает знаки, уступает дорогу и выполняет манёвр. Задачи построены на вопросах официальной теоретической базы (категория «Автомобиль, мотоцикл, мотороллер», источник — [isradrive.info](https://isradrive.info/)).

Технологии: TypeScript + Vite, HTML5 Canvas, физика и управление машиной заимствованы из [parking-game](../parking-game/) (кинематическая велосипедная модель, OBB/SAT-коллизии).

## Спецификация игры

### Игровой цикл

1. Игроку показывается задача: сцена (перекрёсток или участок дороги с знаками, светофорами, другими машинами и пешеходами) и **инструкция** — что нужно сделать (например: «Проедьте перекрёсток налево»).
2. Игрок управляет жёлтой машиной и выполняет манёвр.
3. Задача завершается **успехом** (машина достигла целевой зоны, не нарушив правил) или **провалом** (нарушение или столкновение). После провала показывается пояснение — текст правильного ответа из оригинального вопроса.
4. По нажатию «Дальше» (`N` / `Enter` / кнопка) загружается следующая задача.

Задачи показываются в **случайном порядке** без повторов, пока не исчерпан весь набор; затем набор перемешивается заново. Порядок определяется инъектируемым ГСЧ (для детерминированных тестов).

### Статистика (шапка)

В шапке всегда видны:

- ✅ число успешно пройденных задач;
- ❌ число проваленных;
- **%** — доля успешных от общего числа попыток.

Статистика накапливается за сессию и сохраняется в `localStorage` (сбрасывается кнопкой).

### Условия провала

Задача считается проваленной, если игрок:

- **столкнулся** с машиной, пешеходом или краем дороги (выехал за пределы проезжей части);
- **нарушил приоритет** — въехал на перекрёсток, когда обязан был уступить (машина с приоритетом ещё не проехала / вынуждена тормозить), не пропустил пешехода на переходе;
- **не остановился** полностью перед стоп-линией при знаке «стоп» или красном/жёлтом сигнале светофора;
- **поехал в запрещённом направлении** — манёвр, запрещённый знаком/разметкой, либо выезд не в ту сторону, что указана в инструкции;
- **не уложился во время** (по умолчанию 60 секунд на задачу).

Каждый провал сопровождается сообщением с указанием конкретного нарушения и текстом правильного ответа из вопроса-первоисточника.

### Условия успеха

Машина игрока полностью покинула перекрёсток через **целевой выезд** (соответствующий инструкции), при этом:

- все машины, имевшие приоритет перед игроком, были пропущены (проезд строго в правильном порядке);
- выполнены обязательные остановки (стоп-линия, красный свет);
- не задет никто и ничто.

### Типы задач

Отобрано **94 вопроса**, где ответ можно **показать действием**
(разбивка: 25 приоритет, 8 полосы, 16 светофоры, 38 знаки, 7 пешеходы):

| Тип | Пример вопроса-первоисточника | Что делает игрок |
| --- | --- | --- |
| **Порядок проезда перекрёстка** | «В каком порядке автомобили проедут перекрёсток?» (правило правой руки, главная/второстепенная дорога) | Ждёт своей очереди и проезжает; поехал раньше времени — провал |
| **Предоставление приоритета** | «Кому вы обязаны уступить дорогу?» | Уступает встречным при повороте налево, машинам на главной, пешеходам |
| **Знаки** | «Что означает следующий знак?» (стоп, уступи, въезд запрещён, только направо…) | Действует по знаку: останавливается, уступает, выбирает разрешённое направление |
| **Светофор** | «Что делать при этом сигнале светофора?» | Стоит на красный/жёлтый, едет на зелёный, уступает при мигающем |
| **Пешеходные переходы** | «Как вести себя перед переходом?» | Пропускает пешеходов, не останавливается на переходе |

### Модель задачи (DSL)

Каждая задача — декларативный объект (`src/tasks/*.ts`), полученный вручную из вопроса первоисточника:

```ts
interface Task {
  id: string;                 // номер вопроса isradrive, напр. "0429"
  question: string;           // исходный текст вопроса
  instruction: string;        // что должен сделать игрок
  explanation: string;        // текст правильного ответа (показывается после)
  scene: Scene;
}

interface Scene {
  kind: 'intersection' | 'road';       // перекрёсток или прямой участок
  approaches: Approach[];              // подъезды: N | E | S | W
  signs: { approach: Dir; type: SignType }[];       // знаки на подъездах
  trafficLight?: { approach: Dir; state: LightState }[];
  cars: NpcCar[];                      // машины с намерением и порядком проезда
  pedestrians?: Pedestrian[];
  player: { approach: Dir; goal: Turn };            // откуда едет и куда должен
}

interface NpcCar {
  approach: Dir;              // откуда подъезжает
  turn: Turn;                 // 'left' | 'right' | 'straight'
  order: number;              // место в правильном порядке проезда (0 = первый)
  color: string;              // соответствует цвету в оригинальном вопросе
}
```

Порядок проезда (`order`) задаётся автором задачи по правильному ответу вопроса — движок его **не вычисляет**, а **проверяет**: NPC-машины едут в своём порядке, ждут игрока, когда его очередь; если игрок въезжает на перекрёсток раньше машины с меньшим `order` — провал «не уступил дорогу».

### Управление

Как в parking-game:

| Действие | Десктоп | Мобильный |
| --- | --- | --- |
| Газ | `W` / `↑` | педаль **ГАЗ** |
| Тормоз / задний ход | `S` / `↓` / `Space` | педаль **ТОРМОЗ** |
| Руль | `A`/`D`, `←`/`→` | руль (перетаскивание) |
| Следующая задача | `N` / `Enter` | кнопка **→** |
| Повторить задачу | `R` | кнопка **↻** |

Камера центрируется на машине игрока; масштаб — колесо мыши / щипок.
Для отладки конкретной задачи: `http://localhost:5173/?task=0700`.

### Отрисовка

Canvas 2D, вид сверху. Перекрёсток собирается из подъездов (полосы, осевые линии, стоп-линии, пешеходные переходы — зебра), знаки рисуются пиктограммами у обочины соответствующего подъезда и дублируются крупно в панели инструкции. Машины — как в parking-game (кузов, лобовое стекло, фары/стопы); NPC-машины окрашены как в исходном вопросе (обычно синяя/красная/зелёная/серая).

### Данные вопросов

- Источник: `start.php?d=AB` / `ques.php` на isradrive.info (XML: текст вопроса, 4 варианта, номер правильного, картинка).
- Скрипт `scripts/fetch-questions.sh` выкачивает базу в `data/questions.json` (в репозитории хранится готовый JSON).
- Из базы вручную отобрано ~100 вопросов, каждому написан сценарий в `src/tasks/`.

  npc.test.ts            движение NPC по очереди проезда
  scenario.test.ts       правила: приоритет / стоп / светофор / полосы / зоны
  session.test.ts        статистика и случайный порядок
  tasks.test.ts          валидность и разрешимость всех задач
  helpers/autopilot.ts   «идеальный водитель» для теста разрешимости

## Разработка

Нужен Node.js 18+.

```bash
npm install            # зависимости
npm test               # юнит-тесты (vitest)
npm run test:watch     # тесты в watch-режиме
npm run lint           # ESLint (typescript-eslint, `as`-касты запрещены)
npm run dev            # dev-сервер Vite на http://localhost:5173
npm run build          # проверка типов + сборка в ./docs (GitHub Pages)
npm run preview        # локальный просмотр прод-сборки
```

**TDD.** Тесты пишутся раньше кода: чистые модули (`game/`) покрыты юнит-тестами полностью; рендер и DOM-ввод проверяются вручную. `tasks.test.ts` прогоняет каждую задачу через движок и проверяет, что она разрешима (эталонный «идеальный проезд» завершается успехом).
